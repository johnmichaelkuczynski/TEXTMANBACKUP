NEUROTEXT - MANDATORY REQUIREMENTS
===================================

1. UNCONDITIONAL USER INSTRUCTION COMPLIANCE
   - The app must ALWAYS follow the user's instructions without exception.
   - If user says "WRITE A 100,000 WORD PAPER ON BOTANY", the app complies.
   - If user says "WRITE A 5000 WORD ESSAY ON FREUD", the app generates exactly that.
   - If user puts in a 200,000 word text and says "TURN INTO A 500,000 WORD POEM", the app complies.
   - If user puts in a 4 word poem and says "TURN INTO A 54,000 WORD TREATISE ON BIOLOGY", the app complies.
   - If user enters NO TEXT except instructions like "WRITE A 50,000 WORD PAPER ON FREUD", the app complies.
   - If user says "WRITE A DISSERTATION ON ECONOMICS" with no supplied text, the app complies.
   - NO THRESHOLDS. NO REFUSING. NO EXCEPTIONS.

2. RECONSTRUCTION IS THE MAIN FUNCTION
   - The Reconstruction function is the primary function of NEUROTEXT.
   - It must execute whatever the user instructs.

3. NO INPUT TEXT VALIDATION
   - The app must NOT require input text.
   - If user provides only instructions, that IS the input.
   - The "No Input Text" error must never appear.
   - Instructions alone are sufficient to run.

4. INTELLIGENT INPUT INTERPRETATION
   - If user puts instructions in the main text box, the app treats it as instructions.
   - If user puts text in the instructions box and instructions in the main box, the app figures it out.
   - The app intelligently interprets user intent and complies.

5. APP TITLE
   - The app is called NEUROTEXT, not "Text Model Validator".

6. AUTOMATIC LOGIN
   - In deployment, username JMK must ALWAYS be logged in.
   - No login screen. No errors. JMK is always authenticated.

7. FULL SUITE POPUP BEHAVIOR
   - When Full Suite runs, the popup opens immediately.
   - The popup stays visible through ALL stages: Reconstruction, Objections, Objection-Proof.
   - The popup never closes until the user closes it.
   - If user closes the popup, there must be a way to re-open it.

8. JOB HISTORY NAVIGATION
   - From the History section, user must be able to get back to the main page.
   - Clear Home link or navigation to return to NEUROTEXT.

9. RESUME UNFINISHED PROJECTS
   - From History, clicking an unfinished project must:
     * Load the project back into the main page
     * Restore the original instructions
     * Let the user finish from where it stopped

10. MODIFY FINISHED PROJECTS
    - From History, clicking a finished project must:
      * Load the project back into the main page
      * Restore the original instructions
      * Let the user modify or re-run it

===================================
END OF REQUIREMENTS
